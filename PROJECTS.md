# GitHub Project Map

This account is organized into four engineering programs. Each program has a clear flagship, supporting repositories, public proof, and community-distribution path.

## Project 1 — Raven Scientific Intelligence

**Mission:** Build evidence-linked AI systems for reproducible scientific reasoning and bounded computational biology.

| Repository | Role | Public proof |
|---|---|---|
| [`raven-ai`](https://github.com/simpliibarrii-crypto/raven-ai) | Research core, Evidence Graph, Token Economy, scientific run gates | [Evidence demo](https://simpliibarrii-crypto.github.io/project.html?project=raven-ai) |
| [`simpliibarrii-crypto-raven-biocomputer`](https://github.com/simpliibarrii-crypto/simpliibarrii-crypto-raven-biocomputer) | Bounded biology execution, deterministic tools, artifact receipts | [Sequence demo](https://simpliibarrii-crypto.github.io/project.html?project=raven-biocomputer) |

**Release gate:** reproducible tests, stable task contracts, documented threat boundaries, and at least one independently reproducible use case.

**Community fit:** bioinformatics, AI-for-science, reproducible research, scientific agents.

## Project 2 — Sovereign Agent Runtime

**Mission:** Run, route, inspect, and control agents across desktop and edge hardware without mandatory cloud dependence.

| Repository | Role | Public proof |
|---|---|---|
| [`home-for-ai`](https://github.com/simpliibarrii-crypto/home-for-ai) | Desktop command surface and local orchestration | [Command-room demo](https://simpliibarrii-crypto.github.io/project.html?project=home-for-ai) |
| [`hermes-edge`](https://github.com/simpliibarrii-crypto/hermes-edge) | Device-aware local model and tool routing | [Routing demo](https://simpliibarrii-crypto.github.io/project.html?project=hermes-edge) |
| `jspace-chain` | Private observable control plane for routing, risk, reflection, and audit | [Policy demo](https://simpliibarrii-crypto.github.io/project.html?project=jspace-chain) |

**Release gate:** clean desktop build, measured device benchmarks, explicit fallback policy, and human approval for high-impact actions.

**Community fit:** local AI, on-device LLMs, agent orchestration, AI safety, edge inference.

## Project 3 — Clinical Safety & Care Workflows

**Mission:** Explore consent-aware, tenant-aware clinical AI infrastructure while keeping claims explicitly below certified clinical deployment.

| Repository | Role | Public proof |
|---|---|---|
| [`openclinical-ai`](https://github.com/simpliibarrii-crypto/openclinical-ai) | Consent, signed manifests, tenant boundaries, audit logs, PSW handoff workflow | [Consent-gated demo](https://simpliibarrii-crypto.github.io/project.html?project=openclinical-ai) |

**Release gate:** stronger test coverage, synthetic-data validation, documented privacy review, external clinical feedback, and clear regulatory scoping.

**Community fit after qualification:** open healthcare software, clinical informatics, nursing technology, privacy-preserving AI.

## Repository naming and status rules

- **Original products:** Raven AI, Home for AI, Hermes Edge, OpenClinical AI, Raven BioComputer, JSpace Chain.
- **Portfolio infrastructure:** profile and GitHub Pages repositories.
- **Integration or upstream study:** LangChain NVIDIA.
- Never describe an alpha, preview, study, or private prototype as production-ready.
- Every public claim should link to code, a test, a benchmark contract, or an interactive demonstration.

## Community distribution priorities

1. Hermes Edge → edge AI and local AI communities.
2. Raven AI / Home for AI → open-source agent and agent-safety communities.
3. Raven BioComputer → bioinformatics communities after broader validation.
4. OpenClinical AI → healthcare lists only after production-quality and adoption requirements are met.
